# Requirements refinement

Help the user turn a vague app idea into concrete requirements.

Ask for:

- primary users;
- jobs-to-be-done;
- expected input/output;
- required tools/actions;
- workflows and commands;
- data boundaries/datasources;
- safety risks;
- testable acceptance criteria.

Prefer short iterative interviews. Summarize assumptions before scaffolding or writing files.
